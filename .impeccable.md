# Design Context — Glean

## Users
Personal knowledge workers who want to save articles, notes, videos, books, and products for later retrieval. They value privacy (local-only data), speed, and a calm interface that doesn't demand attention. Context: often on mobile, saving links quickly via share sheet, or reviewing saved content during focused reading time.

## Brand Personality
**Calm, warm, private.**
The interface should feel like a well-lit personal reading room — not a productivity dashboard, not a social feed. There's no urgency, no FOMO, no gamification. Just a quiet place to think.

## Emotional Goal
**Quiet delight.**
Users should feel: "I trust this tool. Nothing feels stressful or demanding. There are subtle moments of thoughtfulness." The experience should reward extended use with small, pleasant surprises — but never demand attention.

## Aesthetic Direction
**Expansive light theme, warm parchment tones, one sharp accent.**
- Light mode only — no dark mode. Warm, paper-like backgrounds (`parchment`, `cream`)
- One opinionated accent color: deep pine/teal (`--ref-pine`) used sparingly
- Typography: Satoshi for UI, DM Sans for body, Newsreader for editorial moments
- Polished but restrained — every element earns its place

## References
**Linear, Arc, Things 3** — Polished, opinionated design with restraint. These apps share:
- Generous whitespace and intentional density
- Sharp, limited accent colors used purposefully
- Understated animations that feel natural, not showy
- Content-first layouts where chrome recedes

## Anti-References
**Explicitly NOT:**
- AI-generic dark mode (no purple gradients, no glowing neon accents, no glassmorphism)
- Busy or feature-heavy (avoid dashboard complexity, no metrics hero sections)
- Corporate startup landing page energy (no aggressive CTAs, no "revolutionary" marketing)

## Design Principles

1. **Calm over clever.** The interface should never feel like it's trying to impress. If an animation is noticed, it's doing too much.

2. **Content is the feature.** The user's saved articles, notes, and links are the purpose. Chrome and UI elements should recede — there are no hero metrics, no usage dashboards.

3. **Warmth, not whimsy.** The design is warm and human, not playful or quirky. Subtle texture, warm tones, personal touches — but no illustration for illustration's sake.

4. **Speed is trust.** Fast interactions, immediate feedback, no loading spinners for simple operations. The interface should feel instant.

5. **Privacy is visible.** Local-only storage is a feature. The UI should communicate "this is yours" — no cloud sync icons, no account management panels, no social features.

## Visual Tokens (Established)

**Colors (OKLCH)**
- Background: `--ref-parchment` (warm paper)
- Surface: `--ref-cream` (near-white cards)
- Accent: `--ref-pine` (deep teal — one opinionated hue)
- Ink: `--ref-ink` (warm near-black)

**Typography**
- UI: Satoshi (400, 500, 700, 900)
- Body: DM Sans (300, 400, 500)
- Editorial: Newsreader (variable)

**Spacing Scale**
- 4px, 8px, 12px, 16px, 24px, 32px, 48px, 64px

**Animation**
- Ease-out curves, not springy
- 180-350ms durations — quick but not instant
- Prefers-reduced-motion respected
